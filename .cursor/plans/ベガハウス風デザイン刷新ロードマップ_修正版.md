# ベガハウス風デザイン刷新ロードマップ（修正版）

## プロジェクト構造の分析

### 現在の構造

- **フレームワーク**: Svelte + Vite
- **主要コンポーネント**: `HeroSection`, `FeaturesSection`, `StoreSection`, `Features2Section`, `Features3Section`, `FAQSection`, `Footer`
- **スタイル**: `styles.css`（グローバル）+ 各コンポーネント内の`<style>`タグ
- **既存画像**: `public/images/` フォルダに `gallery-1.jpg`, `gallery-2.jpg`, `gallery-3.jpg`, `store-exterior.jpg`, `store-interior-1.jpg`, `store-interior-2.jpg`, `logo1.jpg`, `logo2.jpg` が存在

### 修正・新規作成が必要なファイル

**修正が必要なファイル:**

- `src/components/HeroSection.svelte` - メインビジュアルとヘッダーの再構築（既存画像使用）
- `src/components/Footer.svelte` - フッターの再設計
- `styles.css` - グローバルスタイルとデザインシステムの刷新（明朝体メイン、字間設定）
- `index.html` - フォント読み込みの最適化（Noto Serif JP/游明朝を優先）
- `src/App.svelte` - コンポーネント構成の見直し

**新規作成が必要なファイル:**

- `src/components/Header.svelte` - 独立したヘッダーコンポーネント（最小限のナビゲーション、パジネーションドット）
- `src/components/ConceptSection.svelte` - コンセプト・導入テキストエリア（縦書き対応、既存画像使用）
- `src/components/PropertyGallery.svelte` - **物件一覧・検索コーナー**（ベガハウスの実例集風の美しいカード形式、既存画像使用）
- `src/components/QuickLinksSection.svelte` - 4カラムのコンテンツナビゲーション

## フェーズ別実装計画

### フェーズ1: 全体の共通設定（デザインシステム構築）

**目標**: 信頼感と落ち着きを表現するデザイントークンの確立（明朝体メイン、字間設定）

**作業内容:**

1. **デザイントークンファイル作成** (`src/styles/design-tokens.css`)

- カラーパレット: 白基調、アクセントカラー（控えめなグレー、ベージュ系）
- **タイポグラフィ**: **明朝体（Noto Serif JP または 游明朝）をメイン**に設定
  - メインフォント: `'Noto Serif JP', 'Yu Mincho', '游明朝', 'Hiragino Mincho ProN', 'MS PMincho', serif`
  - サブフォント（本文用）: `'Noto Sans JP', 'Meiryo UI', 'Hiragino Kaku Gothic ProN', sans-serif`
- **字間（letter-spacing）**: 高級感のある字間設定
  - 見出し: `0.05em - 0.1em`
  - 本文: `0.02em - 0.05em`
  - 縦書きテキスト: `0.1em - 0.15em`
- スペーシングシステム: 8px基準のグリッド（8, 16, 24, 32, 48, 64, 96, 128px）
- ブレークポイント: mobile(768px), tablet(1024px), desktop(1440px)

2. **グローバルスタイル更新** (`styles.css`)

- リセットCSSの最適化
- ベースラインの統一（line-height: 1.6-1.8）
- フォントレンダリングの最適化（-webkit-font-smoothing, text-rendering）
- **明朝体をデフォルトフォントとして設定**

3. **フォント設定** (`index.html`)

- **Noto Serif JP または 游明朝を優先的に読み込み**
- フォント読み込みの最適化（preload, display: swap）

**デザインの要点:**

- 余白: セクション間は96-128px、要素間は24-48px
- フォントサイズ: 見出し（32-48px、明朝体）、本文（16-18px、明朝体またはゴシック体）
- **字間**: 見出し0.05-0.1em、本文0.02-0.05em、縦書き0.1-0.15em
- カラー: 背景は純白（#FFFFFF）、テキストは濃いグレー（#1A1A1A）、アクセントは控えめ

## デザインの要点まとめ

### 清潔感と高級感を出すためのCSSポイント

1. **余白の取り方**

- セクション間: 96-128px（デスクトップ）、64-96px（モバイル）
- 要素間: 24-48px
- コンテナの最大幅: 1200-1440px、パディング: 32-48px

2. **タイポグラフィ（重要）**

- **メインフォント: 明朝体（Noto Serif JP または 游明朝）**
- 見出し: 32-48px（明朝体）、行間1.4-1.6、**字間0.05-0.1em**
- 本文: 16-18px（明朝体またはゴシック体）、行間1.6-1.8、**字間0.02-0.05em**
- 縦書きテキスト: 24-32px（明朝体）、行間1.8-2.0、**字間0.1-0.15em**
- フォントレンダリング: -webkit-font-smoothing: antialiased

3. **カラーパレット**

- 背景: #FFFFFF（純白）
- テキスト: #1A1A1A（濃いグレー）
- アクセント: #8B7355（ベージュ系）または #6B6B6B（グレー）
- ボーダー: #E5E5E5（薄いグレー）

4. **影とエフェクト**

- ボックスシャドウ: 控えめ（0 2px 8px rgba(0,0,0,0.08)）
- ホバーエフェクト: 軽い拡大（transform: scale(1.02)）または色変化
- トランジション: 0.3s ease

5. **縦書きテキスト**

- writing-mode: vertical-rl（右から左）
- text-orientation: upright（文字の向き）
- フォント: 明朝体（Noto Serif JP 等）
- **字間: 0.1-0.15em**

6. **画像の使用方針**

- **既存画像を優先使用**（`public/images/`フォルダ内）
- プレースホルダー画像は使用しない
- 高品質な写真を使用し、object-fit: coverで統一

1. **最優先**: フェーズ1（デザインシステム、明朝体メイン、字間設定）→ フェーズ2（ヘッダー）
2. **次**: フェーズ3（メインビジュアル・コンセプト、既存画像使用）

各フェーズは独立して実装可能ですが、フェーズ1のデザインシステムは他のフェーズの基盤となるため、最初に完了させることを推奨します。
